壹 WHAT IS IT

它是什么

市面金铲铲工具只查 阵容表 —— 告诉你"这套强"。这个项目两头补：你玩的时候有人提醒你该选哪个增强 · 要不要升人口 · 装备合给谁；你打完了自动出一份带量化反事实的复盘报告。

本质是同一套感知层 · 两条产品路径。OBS 虚拟摄像头吃实时 · mp4 录屏吃赛后 · 进共享的 frame_monitor / OCR / VLM / S17 RAG · 出一致的 WorldState · 分叉到实时建议或整局复盘。

叁 ARCHITECTURE

技术架构

两条产品路径共用一套感知层。数据源二选一 · 感知层统一 · 分析层按需分叉。设计目标：让专精模型做专精事 —— 各层独立失败互不污染。

层 · 代码 做什么 为什么不全交给 VLM
OCR 层
ocr_client.py · PaddleOCR		 读 HP · 金币 · 等级 · 连胜数的中文数字 实测 Qwen2.5-VL 读 HP 常错 1-2 · 数据分析致命。OCR 稳定 99%+ · 失败模式也可预测。
CV 层
arrow_finder.py · OpenCV		 找装备图标 · 棋子位 · 高亮 UI 元素中心 颜色阈值 + 形状匹配 30 行代码够用 · 比 VLM 快一个数量级 · 也不会"幻觉"出不存在的图标。
VLM 层
vlm_client.py · Qwen3-VL-8B/4B-FP8		 羁绊语义 · 对手阵容分类 · 教程引导文字 把 VLM 留给它真正擅长的语义任务 · 不让它读它做不好的数字。显存紧张就降到 4B-FP8。
分析层
llm_analyzer.py · decision_llm.py		 实时 6 类决策 / 赛后整局合成 · 量化反事实 需要 S17 版本知识 RAG · 默认本地 vLLM 零 API 成本。赛季更新只换 RAG · 不重训模型。
为什么混合 · 不押单一大模型

实测 Qwen2.5-VL-7B 看 HP 经常错 1-2。换 Qwen3-VL-8B-FP8 精度够 · 但 16 GB 显存装不下 + MuMu 模拟器 · 降到 Qwen3-VL-4B-FP8 是显存敏感方案。最终 —— 专精模型做专精事 · 各层独立失败 —— 比押注单一大 VLM 稳得多。这个架构判断本身就值得写进简历。

伍 COMPLIANCE

为什么腾讯感知不到

这不是"代打换皮"。从数据源到执行面都刻意远离游戏进程 —— 腾讯反外挂看不到异常 · 简历能正面写。

只读屏
read-only capture
数据源是 OBS 虚拟摄像头 · 本质是另一路视频流。不挂钩游戏进程 · 不读游戏内存 · 不发任何输入指令。所有操作仍由玩家本人完成。
本地推
local vLLM first
感知 / 分析 / 决策共用一个本地 vLLM 实例 · 默认零 API 成本 · 零网络依赖。想接云端写一个 CloudLLMAnalyzer 替换就行 · 接口不变。
不碰 Android
no adb / no hook
数据源阶段曾经用过 ADB 截屏 —— 现已整层替换为 OBS 虚拟摄像头。不接 Android 侧 · 不用 scrcpy · 不用 accessibility · 模拟器认为自己只是在被录。
C O M P L I A N C E S T A C K
  • OBS Virtual Camera 只摄不控 · 和直播软件同类
  • Python 端只拿视频流 · 不触 MuMu 进程
  • PyQt overlay click-through · 不拦截点击
  • 无 ADB / scrcpy / Accessibility / Frida
  • 无输入注入 · 最终操作仍是玩家点屏幕
  • 不做自动化 · 不做点击 · 不做寻路
陆 HOW TO RUN

怎么跑通

最短路径 —— 不需要 OBS · 不需要 Windows · 在 WSL 里跑通全链路 · 50 秒内出一份 HTML 复盘。想上实时 coach 再走 Windows 四终端。

赛 后 复 盘
WSL · one-liner · end-to-end smoke
# 装依赖
pip install -r requirements.txt

# 端到端 smoke · 50s 出 HTML 复盘
python3 scripts/e2e_smoke.py
# → /tmp/e2e_smoke_report.html (≈ 18 KB)

# 或者从真 mp4 录屏抽帧分析
python scripts/analyze.py \
  --video match.mp4 --every-s 5 \
  --vlm real --llm real \
  --out report.md

vLLM 可用就真跑 5 类 decisionLLM + 整局合成；不可用自动 fallback · pipeline 仍然跑通。

实 时 教 练
Windows 原生 · 四终端 · OBS 前置
# T1 · WSL · 启 vLLM（感知+分析+决策共用）
python -m vllm.entrypoints.openai.api_server \
  --model /path/to/Qwen3-VL-4B-FP8 --port 8000

# T2 · advice 广播服务
python -m src.advice_server --port 8765

# T3 · Windows · OBS virtual cam + tick
python -m src.live_tick --fps 2 \
  --advice-server http://localhost:8765

# T4 · Windows · PyQt overlay 半透明卡片
python -m src.overlay_ui \
  --ws-url ws://localhost:8765/ws/advice

前置：OBS Studio · Window Capture 抓 MuMu · Start Virtual Camera。overlay 自动找 MuMu 主窗口并贴边 · 决策点触发淡入 · 8 秒淡出。
